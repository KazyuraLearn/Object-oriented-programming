Имена:
	1. Для именования экземпляров классов использовать lowerCamelCase.
	2. Для именования методов, свойств, классов использовать UpperCamelCase.
	3. Для именования интерфейсов использовать UpperCamelCase с венгерской нотацией (например: interface IEnumerable).
	4. Полные имена, если они слишком длинные для одной строки, можно разбить после точки (.).
	
Расположение:
	1. Использование табуляции для отсупов.
	2. Запись одного оператора в строке. В редких случаях запись двух операторов, если они логически дополняют или продолжают друг друга (например: file.Close(); writer.Close();).
	3. Между определениями методов или свойст добавляется пустая строка.
	4. Допускается не использовать скобки после операторов ветвления или цикла, если за ними идёт один оператор. Правило также верно для вложенных операторов.
	5. Скобки всегда располагаются с новой строки, за исключением пустых акссесоров и отсутсвия тела (например: public int T { get; private set; } или class T { }).

Комментарии:
	1. Комментарий размещается на отдельной строке, а не в конце строки кода.
	2. Текст комментария начинается с заглавной буквы и завершается точкой.
	3. Между разделителем комментария и текстом комментария вставляется один пробел.

Оператор using и try-catch-finally:
	1. Рекомендуется использовать оператор using, если в finally содержится только Dispose.

Оператор ветвления:
	1. Рекомендуется использовать тернарный оператор вместо if-else где это возможно и не вредит читаемости.
